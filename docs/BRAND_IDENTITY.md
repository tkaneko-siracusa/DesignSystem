# Polastack Design System - ブランドアイデンティティ

> 本ドキュメントは、Polastackプラットフォーム全体の視覚的アイデンティティを定義する。

---

## 1. ブランドカラー

### プラットフォームカラー（Polastack共通）

| トークン | 用途 | 値 |
|---------|------|-----|
| `primary-50` | 薄い背景・ホバー | #eff6ff |
| `primary-100` | サブ背景 | #dbeafe |
| `primary-200` | ボーダー・非アクティブ | #bfdbfe |
| `primary-300` | リング・フォーカス | #93c5fd |
| `primary-400` | アイコン（非アクティブ） | #60a5fa |
| `primary-500` | メインカラー | #3b82f6 |
| `primary-600` | ホバー・アクティブ | #2563eb |
| `primary-700` | プレス状態 | #1d4ed8 |
| `primary-800` | ダーク背景 | #1e40af |
| `primary-900` | 最も濃い背景 | #1e3a8a |
| `primary-950` | ダークモードテキスト | #172554 |

### セマンティックカラー

| トークン | 用途 | 色相 |
|---------|------|------|
| `success` | 成功・有効・完了 | Green (emerald系) |
| `warning` | 警告・注意 | Amber |
| `error` | エラー・危険・削除 | Red |
| `info` | 情報・ヘルプ | Blue |

### ニュートラルカラー

Zinc系を採用。テーブル行・ボーダー等の微妙なコントラスト差を表現できる広いグレースケール。

| トークン | 用途 |
|---------|------|
| `neutral-50` ～ `neutral-950` | テキスト、背景、ボーダー全般 |
| `text-primary` | 本文テキスト（neutral-900） |
| `text-secondary` | 補足テキスト・ラベル（neutral-500） |
| `text-tertiary` | プレースホルダー（neutral-400） |
| `text-disabled` | 非活性要素（neutral-300） |

### プロダクト別テーマの許容範囲

| カテゴリ | カスタマイズ可否 | 備考 |
|---------|---------------|------|
| **ブランドカラー（primary-*）** | 可 | プロダクト独自のプライマリカラーに変更可能 |
| **セマンティックカラー** | 不可 | 赤=エラー、緑=成功は全プロダクト共通 |
| **ニュートラルカラー** | 限定的 | 背景・テキストの微調整は可 |
| **タイポグラフィ** | 限定的 | フォントファミリーは変更可。サイズスケールは共通 |
| **スペーシング** | 不可 | 全プロダクト共通 |
| **コンポーネント構造** | 不可 | Buttonの形状やDialogの構造は共通 |

## 2. タイポグラフィ

### フォントファミリー

- **本文（欧文）**: Inter - 小サイズでの可読性に優れる
- **本文（和文）**: Noto Sans JP - 高品質な日本語フォント
- **等幅フォント**: JetBrains Mono - コード表示用

### フォントスケール

BtoB業務アプリの高密度表示に最適化。base=14px（Ant Design/Atlassianと同等）。

| トークン | サイズ | 行高 | 用途 |
|---------|-------|------|------|
| `text-xs` | 12px | 16px | テーブルセル、注釈 |
| `text-sm` | 13px | 20px | 本文（高密度モード） |
| `text-base` | 14px | 20px | 本文（標準モード） |
| `text-lg` | 16px | 24px | 小見出し |
| `text-xl` | 18px | 28px | セクション見出し |
| `text-2xl` | 20px | 28px | ページタイトル |
| `text-3xl` | 24px | 32px | ダッシュボード見出し |

### フォントウェイト

| ウェイト | 用途 |
|---------|------|
| 400 (Regular) | 本文 |
| 500 (Medium) | ラベル、サブヘッダー |
| 600 (Semibold) | 見出し |
| 700 (Bold) | 強調、ページタイトル |

## 3. アイコン

### ガイドライン

- **ライブラリ**: Lucide React（アウトラインスタイルで統一）
- **線の太さ**: 1.5px（Lucideデフォルト）
- **塗りつぶし**: 基本不使用。選択状態・アクティブ状態の表現に限定

### アイコンサイズ

| サイズ | px | 用途 |
|-------|-----|------|
| `xs` | 12px | インラインテキスト内 |
| `sm` | 16px | ボタン内、テーブルセル |
| `md` | 20px | ナビゲーション、標準アイコン |
| `lg` | 24px | ページヘッダー |
| `xl` | 32px | 空状態、エンプティステート |

## 4. ボイス&トーン（UIテキスト）

### 基本方針

- **敬語レベル**: 丁寧語（〜です、〜ます）を基本。謙譲語・尊敬語は不使用
- **文体**: 簡潔で明確。曖昧な表現を避ける
- **漢字/ひらがな比率**: 読みやすさ優先。難しい漢字はひらがなに開く

### テキストパターン

| 種類 | ガイドライン | 例 |
|------|-----------|-----|
| **ボタンラベル** | 動詞で始める。2〜4文字 | 保存、削除、追加 |
| **確認ダイアログ** | 何が起きるかを明確に | 「このレコードを削除しますか？この操作は取り消せません。」 |
| **エラーメッセージ** | 原因 + 対処法 | 「メールアドレスの形式が正しくありません。例: user@example.com」 |
| **成功メッセージ** | 完了した内容を簡潔に | 「レコードを保存しました」 |
| **空状態** | 状態の説明 + アクション | 「データがありません。「追加」ボタンで作成してください。」 |

## 7. アニメーション

### 基本方針

- アニメーションは**機能的**であること。装飾的なアニメーションは最小限
- `prefers-reduced-motion` を尊重し、動きを減らすモードに対応

### トランジション速度

| トークン | 値 | 用途 |
|---------|-----|------|
| `duration-fast` | 100ms | ホバー、フォーカス |
| `duration-normal` | 200ms | 開閉、表示/非表示 |
| `duration-slow` | 300ms | ページ遷移、大きなレイアウト変化 |

### イージング

| トークン | 値 | 用途 |
|---------|-----|------|
| `ease-default` | `cubic-bezier(0.4, 0, 0.2, 1)` | 一般的なトランジション |
| `ease-in` | `cubic-bezier(0.4, 0, 1, 1)` | 要素の退出 |
| `ease-out` | `cubic-bezier(0, 0, 0.2, 1)` | 要素の出現 |
